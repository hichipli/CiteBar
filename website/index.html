Install

Quick install and first-launch fixes

Download it from GitHub, move it to Applications, paste a Scholar profile URL, and you're up. Then you can monitor citations, open tracked profiles fast, and keep recent changes visible from the menu bar.

Install in 4 clear steps

  1. 1

    Download CiteBar

    Get the latest free universal DMG from GitHub Releases. DMG is the only official distribution format.

  2. 2

    Move to Applications

    Open the DMG and drag CiteBar.app into /Applications.

  3. 3

    Fix first-launch security block (if prompted)

    Press Cmd + Space, open Terminal, then run this command:

    xattr -cr /Applications/CiteBar.app

  4. 4

    Launch and configure

    Open CiteBar, paste a full Google Scholar profile URL, and start tracking in under a minute. Current releases store settings and citation history locally on your Mac.

Auto-update compatibility: if you are on 1.3.x or 1.4.1, do one manual DMG upgrade first. After you are on 1.4.4+, automatic updates should work normally.

First-launch troubleshooting

If macOS blocks launch, use these fixes in order. Direct open-source downloads can trigger a warning even when the app is safe to inspect and the code is public.

  • "CiteBar is damaged" usually means macOS quarantine, not app corruption. This is expected for direct, non-notarized open-source downloads.
  • "Cannot verify developer": right-click CiteBar.app, choose Open, then confirm Open again.
  • Still blocked? Run sudo xattr -cr /Applications/CiteBar.app and then sudo chmod -R 755 /Applications/CiteBar.app.
  • System requirement: macOS 13.0+ on Intel or Apple Silicon.

FAQ

Common install, privacy, and update questions

Is CiteBar open source and free to use?

Yes. CiteBar is open source on GitHub under the MIT License, and it is free to use.

How does CiteBar handle privacy and data?

Current releases store settings and citation history locally on your Mac and do not send telemetry to CiteBar servers.

Why does macOS say "CiteBar is damaged and can't be opened"?

This is usually a quarantine warning for non-notarized downloads. Run xattr -cr /Applications/CiteBar.app, then open the app again.

What if macOS says "Cannot verify developer"?

In Applications, right-click CiteBar.app, choose Open, then click Open in the dialog.

I am on 1.3.x or 1.4.1 and auto-update fails. What should I do?

Manually install the latest DMG once. After upgrading to 1.4.4+, in-app automatic updates should work normally.

What should I paste when adding a Scholar profile?

Paste the full public Google Scholar profile URL. It can be your own profile or any other public profile you want to track. Display name is optional.

Where should I report bugs or request features?

Use GitHub Issues. Include your macOS version, installed CiteBar version, and screenshots if possible.
